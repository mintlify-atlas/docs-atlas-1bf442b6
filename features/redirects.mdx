## Overview

The **Redirects Manager** provides a simple interface to create and manage URL redirects without touching server configuration files. Supports 301 permanent and 302 temporary redirects with wildcard pattern matching.

<Info>
  Proper redirects preserve SEO value when moving or restructuring content, preventing 404 errors and maintaining link equity.
</Info>

## Redirect Types

<AccordionGroup>
  <Accordion title="301 Permanent Redirect">
    **Use when:** Content has permanently moved to a new URL
    
    **SEO Impact:**
    - Passes 90-99% of link equity to new URL
    - Tells search engines to update their index
    - Should be used for permanent changes
    
    **Examples:**
    - Site restructure: `/old-section/*` → `/new-section/`
    - Page rename: `/contact-us` → `/contact`
    - Domain change: All pages to new domain
  </Accordion>
  
  <Accordion title="302 Temporary Redirect">
    **Use when:** Content is temporarily at a different URL
    
    **SEO Impact:**
    - Does NOT pass link equity
    - Tells search engines the change is temporary
    - Original URL stays in index
    
    **Examples:**
    - A/B testing different landing pages
    - Seasonal promotions
    - Maintenance pages
    - Limited-time campaigns
  </Accordion>
</AccordionGroup>

## Implementation Details

### Redirects Class

```php title="includes/class-geoai-redirects.php" showLineNumbers
class GeoAI_Redirects {
    private static $instance = null;

    public static function get_instance() {
        if ( null === self::$instance ) {
            self::$instance = new self();
        }
        return self::$instance;
    }

    private function __construct() {
        add_action( 'template_redirect', array( $this, 'maybe_redirect' ), 1 );
    }

    public function maybe_redirect() {
        if ( is_admin() ) {
            return;
        }

        $redirects    = get_option( 'geoai_redirects', array() );
        $request_path = $_SERVER['REQUEST_URI'];

        foreach ( $redirects as $redirect ) {
            if ( empty( $redirect['from'] ) || empty( $redirect['to'] ) ) {
                continue;
            }

            $from = $redirect['from'];
            $to   = $redirect['to'];
            $type = $redirect['type'] ?? 301;

            if ( $this->matches( $request_path, $from ) ) {
                wp_redirect( $to, $type );
                exit;
            }
        }
    }
}
```

### Pattern Matching

```php title="includes/class-geoai-redirects.php" showLineNumbers startLineNumber="55"
private function matches( $request, $pattern ) {
    $pattern = str_replace( '*', '(.*)', $pattern );
    $pattern = '#^' . $pattern . '$#';
    
    return (bool) preg_match( $pattern, $request );
}
```

## Wildcard Patterns

Use `*` to match any characters:

<CodeGroup>

```text Match All in Directory
From: /blog/*
To:   /articles/

Matches:
✓ /blog/post-1
✓ /blog/post-2  
✓ /blog/category/tech
```

```text Match File Extensions
From: *.php
To:   /

Matches:
✓ /old-page.php
✓ /contact.php
✗ /page (no .php)
```

```text Match Middle Pattern
From: /docs/*/download
To:   /downloads/

Matches:
✓ /docs/pdf/download
✓ /docs/2024/download
✗ /docs/download (missing middle)
```

```text Match Query Strings
From: /page*
To:   /new-page

Matches:
✓ /page
✓ /page?id=123
✓ /page?utm_source=google
```

</CodeGroup>

## Redirect Chains

<Warning>
  Avoid redirect chains. Each additional redirect adds ~200-500ms latency and dilutes SEO value.
</Warning>

**Bad:**
```text
/old-page → /renamed-page → /final-page
```

**Good:**
```text
/old-page → /final-page
/renamed-page → /final-page
```

### Server-Level Alternatives

For very high-traffic sites, consider server-level redirects:

<Tabs>
  <Tab title="Apache (.htaccess)">
    ```apache title=".htaccess"
    # Single redirect
    Redirect 301 /old-page /new-page
    
    # Directory redirect
    RedirectMatch 301 ^/blog/(.*) /articles/$1
    
    # External redirect  
    Redirect 301 /docs https://docs.example.com
    ```
  </Tab>
  
  <Tab title="Nginx">
    ```nginx title="nginx.conf"
    # Single redirect
    location /old-page {
        return 301 /new-page;
    }
    
    # Pattern redirect
    location ~ ^/blog/(.*) {
        return 301 /articles/$1;
    }
    
    # External redirect
    location /docs {
        return 301 https://docs.example.com;
    }
    ```
  </Tab>
</Tabs>

## Programmatic Management

Manage redirects via code:

```php
// Get current redirects
$redirects = get_option( 'geoai_redirects', array() );

// Add new redirect
$redirects[] = array(
    'from' => '/old-page',
    'to'   => '/new-page',
    'type' => 301,
);

// Save
update_option( 'geoai_redirects', $redirects );
```

### Bulk Import

```php
// Import from CSV
$csv_data = array(
    array( '/old-1', '/new-1', 301 ),
    array( '/old-2', '/new-2', 301 ),
    array( '/promo', '/sale', 302 ),
);

$redirects = array();
foreach ( $csv_data as $row ) {
    $redirects[] = array(
        'from' => $row[0],
        'to'   => $row[1],
        'type' => $row[2],
    );
}

update_option( 'geoai_redirects', $redirects );
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Redirect not working">
    **Check:**
    - From URL matches exactly (case-sensitive)
    - Include leading slash: `/page` not `page`
    - Redirect is saved (click Save Changes)
    - Clear browser cache
    - Test in incognito mode
  </Accordion>
  
  <Accordion title="Redirect loop">
    **Cause:** Circular redirect (A → B → A)
    
    **Solution:**
    - Check both GEO AI and other redirect sources
    - Verify destination URL is correct
    - Remove conflicting redirects
  </Accordion>
  
  <Accordion title="Wildcard not matching">
    **Common issues:**
    - Pattern must match full path: `/blog/*` not `blog/*`
    - Use `*` not `%` or other wildcards
    - Test pattern with specific examples
  </Accordion>
  
  <Accordion title="Too many redirects error">
    **Cause:** Redirect chain or loop
    
    **Solution:**
    - Remove intermediate redirects
    - Redirect directly to final destination
    - Check server-level redirects (htaccess, nginx)
  </Accordion>
</AccordionGroup>

## SEO Impact

### Link Equity Transfer

| Redirect Type | Link Equity Passed | Indexing Behavior |
|---------------|-------------------|-------------------|
| 301 Permanent | ~90-99% | New URL indexed, old URL removed |
| 302 Temporary | 0% (minimal) | Old URL stays indexed |
| 307 Temporary | 0% (minimal) | Old URL stays indexed |

<Tip>
  Google treats 301, 302, and 307 similarly now, but 301 is still best practice for permanent moves.
</Tip>

### Time to Re-index

After creating 301 redirects:
- Google discovers redirect: 1-7 days
- Link equity transfers: 2-4 weeks
- Old URL drops from index: 4-8 weeks
